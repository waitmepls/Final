ROS Master ทำหน้าที่เป็น ตัวกลาง server สำหรับ node หลายๆ node เชื่อมต่อกัน และส่ง message หากันได้

Ros Topics คือ ข้อความจะถูกส่งผ่านระบบการขนส่งที่มีการ publish / subscribe
โหนดที่ส่งข้อความจะเรียกว่า publishing ชื่อที่ระบุใน Topic จะใช้เพื่อระบุเนื้อหาของ
ข้อความ โหนดที่สนใจในข้อมูลบางประเภทจะทำการ subscribe หัวข้อที่ต้องการ อาจมี
การ publish / subscribe หลายรายพร้อมกันสำหรับหัวข้อเดียวและโหนดเดียวอาจ
publish / subscribe ได้หลาย Topics

Ros Service คือ การใช้รูปแบบ Request / reply กำหนดการส่งข้อความ โดยสร้าง
ข้อความหนึ่งสำหรับการร้องขอและอีกหนึ่งสำหรับการตอบกลับ เพื่อส่งข้อความที่ร้องขอไป
ยังโหนดที่ร้องขอ


ความแตกต่างระหว่าง topic กับ message
Message คือ node แต่ละตัวจะคุยกันผ่าน message, โดย message ก็
อาจจะเป็นชนิดของตัวแปร เช่น integer, floating point, boolean หรืออาจจะเอามา
ประกอบกันเป็น structure เช่นประกอบเป็น array หรือประกอบรวม int, float, string
เข้าด้วยกัน

Topic คือ publisher node จะ publish message ของตัวเองขึ้นไปที่
topic แล้วจะมี subscriber node มารอรับ message ที่ส่งมาทาง topic ที่มีชื่อเดียวกัน

ความแตกต่างของ ROS Topic และ ROS Service
Topic คือ 
โดยมีรูปแบบอำนวยความสะดวกในการสื่อสารแบบอะซิงโครนัส หมายความว่า
โหนดผู้เผยแพร่ไม่จำเป็นต้องรอให้ผู้สมัครสมาชิกได้รับข้อความ
ข้อความที่เผยแพร่ในหัวข้อจะถูกส่งไปยังสมาชิกทั้งหมด

Service คือ บริการใน ROS มีกลไกการตอบสนองคำขอสำหรับการสื่อสาร
อำนวยความสะดวกในการสื่อสารแบบซิงโครนัส หมายความว่าโหนดไคลเอ็นต์ส่งคำ
ขอและรอจนกว่าจะได้รับการตอบกลับจากโหนดเซิร์ฟเวอร์
